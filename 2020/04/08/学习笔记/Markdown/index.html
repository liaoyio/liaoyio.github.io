    Markdown基本语法总结
    发表于2020-04-08|更新于2020-04-09
    |阅读量:

    序言:
    搭建个人博客的时候，耗时数周，在网上到处查阅资料，零基础啥也不是，可依旧沉迷其中不亦乐乎。三个月没有写博客了，这可能就是惰性吧，学的知识点总是记不住，现在的状态是：看的越多，懂的越少！为什么这么说呢，每次心血来潮在浏览前端知识的时候，三秒就各种知识点给淹没了，根本就不知道哪些是我现在能吸收的，哪些是我现阶段不应该深入的。一看就会，一写就废，我把我的这种状态定义为不懂装懂，究其根本就是看的太多，写的太少，所以要改掉这样毛病，把被动学习变成主动学习，接下来的日子，多写写，即使是写一些学习工作中遇到的坑也是好的，没事翻出来看看，还可以加深印象，好了，废话到处！

    起因：

    写博客的时候总是忘记一些语法，导致上传后各种滑稽现场(表单不连贯，代码块和文本合并，导航栏杂乱等)，还是记重新总结记录一下吧，方便以后翻阅。本文是我学习写Markdown文件的一些笔记，仅供参考！

    十、转义

    转义的话要用到反斜杠，Markdown中，以下字符支持使用反斜线转义：

    效果如下:

    \ 反斜线 //与JavaScript转义一致

    * 星号
    _ 下划线
    { } 大括号(分左右)
    [ ] 中括号(分左右)
    ( ) 小括号(分左右)
    # 井号
    + 加号
    - 减号（连字符）
    . 句点
